# EMAIL DRAFT GATE: run BEFORE pasting any email draft in chat

Applies to: pb-email, pb-email-write, va-email, va-email-write, cc-email, rr-email, any skill that outputs an email draft.

## The failure this kills
2026-06-10 quiz-email session: four rules that ALREADY EXISTED (specific-named-technique, swipe-file pacing, no-oscillation, batch diversity) were all in context via the angle-ledger hook and still got broken at draft time. Knowledge was present; execution wasn't. Prose reminders don't fix this. A checklist that produces a visible artifact does, because skipping it is detectable by Timo at a glance.

## THE GATE: 4 checks, then a 1-line proof block

1. **PACING (read, don't recall).** Read 2-3 actual emails from `voc/emails/swipe-file/raw/dimitri-fantini-drums/` THIS session before drafting. Match the observed rhythm: mostly 1-2 sentence paragraphs with air, an occasional 3-sentence unit, a single-line punch after a dense moment. BOTH walls AND one-sentence-fragment lists are banned. Never draft from a remembered description of the rhythm.

2. **FLOW (read every seam aloud).** A question's grammar must match its answers: action question ("what should you do?") gets action answers; cause question gets cause answers. Segues must be continuous; no orphan paragraphs that restate or re-ask. If a draft has a quiz/list device, read straight through it as one thought and check it doesn't clunk.

3. **SPECIFICITY (a move, not a diagnosis).** The mechanism taught must be a do-able, NAMED Harrison technique, verbatim-sourced from a transcript you grepped this session (know the file). "Your embouchure was never built for you" is a diagnosis = soft cock. "Say 'Sim' and place the mouthpiece from the top lip down" is an answer. The reader must leave with something they can do in the mirror tonight.

4. **MIDDLE, NOT PENDULUM.** After any structure correction from Timo, re-read the last 2 corrections in the conversation and synthesize toward the middle. Never flip to the opposite extreme (wall to fragment list, soft to shock, etc.).

## Proof block (mandatory)
Print ONE line directly above every draft pasted in chat:

`GATE: pacing=<swipe files read> | flow=Q/A seams checked | technique=<named move @ source file>`

No proof line = gate didn't run = draft is not paste-able. Timo can reject on sight.
