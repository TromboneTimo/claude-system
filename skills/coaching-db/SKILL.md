# Coaching Database Skill

Per-project Voice-of-Customer (VOC) database. Raw transcripts are ground truth. Quote bank is the retrieval layer. Voice bank is the fast-reference overlay. Fresh-angle mining pulls new quotes from the same raw corpus under different lenses.

## Subcommand routing

When the user invokes `/coaching-db <sub>`, route as follows:

| Subcommand | Action |
|---|---|
| `init` | Ask setup questions. Create `./voc/` skeleton in current workspace. Write `voc/config.yaml`. |
| `ingest <path-or-urls>` | Pull/normalize raw transcripts into `voc/raw/<source-type>/`. YouTube URLs via `scripts/fetch-youtube-transcript.sh`. VTT/SRT files via `scripts/vtt-to-markdown.py`. |
| `extract` | Spawn parallel agents to read all raw files without a corresponding quote extraction, output verbatim quotes with YAML tags to `voc/quotes/all-quotes.jsonl`. |
| `mine <angle>` | Spawn a fresh agent that re-reads ALL raw files under a specific lens (e.g., `identity-loss`, `what-they-tried`, `mouthpiece-objections`), pulls NEW quotes tagged with the angle, appends to `voc/quotes/all-quotes.jsonl`. De-duplicates against existing quotes. |
| `synthesize` | Run `scripts/build-voice-bank.py` to regenerate `voc/personas/voice-bank.md` from the JSONL. |
| `status` | Report counts: raw files per source type, total quotes, funnel distribution, pain-point distribution, gaps flagged. |

## Per-project database layout

```
<project-root>/voc/
├── config.yaml
├── raw/                           IMMUTABLE. Never edit.
│   ├── sales-calls/
│   ├── testimonials/
│   ├── youtube-transcripts/
│   ├── youtube-comments/
│   ├── facebook-comments/
│   └── dms/
├── quotes/
│   └── all-quotes.jsonl           One quote per line. Master index.
└── personas/
    ├── prospect-psychology.md     Optional inherited distilled overlay.
    ├── voice-bank.md              Generated by `synthesize`.
    └── winning-phrases.md         Optional language-that-converted file.
```

## Quote entry schema

Every quote written to `all-quotes.jsonl` must match this schema. See `templates/quote-entry.md` for the canonical example.

```json
{
  "quote": "exact verbatim text",
  "speaker_type": "prospect | customer | commenter",
  "speaker_name": "inferred first name or 'unknown'",
  "source_file": "raw/sales-calls/2026-03-23_salescall_michael_call-0.md",
  "source_timestamp": "00:14:22",
  "pain_point": ["range", "endurance"],
  "emotional_trigger": ["frustration"],
  "funnel_stage": "TOFU | MOFU | BOFU",
  "use_for": ["hook", "ad-copy"],
  "mining_angle": "before-state",
  "confidence": "high | medium | low"
}
```

## Anti-hallucination gates (MANDATORY)

Applied every run. See `references/anti-hallucination.md` for the full rationale.

1. **Verbatim only.** Never paraphrase. Copy exact words from the raw transcript. Light punctuation cleanup is fine. Word substitution is not.
2. **Traceable or reject.** Every quote needs `source_file` + `source_timestamp`. If you cannot produce both, do not extract the quote.
3. **Prospect vs customer matters.** Sales-call speakers default to `prospect`. Testimonial speakers default to `customer`. Commenters default to `commenter`. Never conflate.
4. **Auto-transcripts lie.** When `transcript_source` is auto-captions (Zoom, YouTube), flag every extracted quote with `confidence: medium` at most. Reserve `high` for human-reviewed transcripts only.
5. **Unknown speaker is valid.** If a VTT has no speaker diarization or the name can't be inferred, set `speaker_name: "unknown"`. Never guess.

## Routing logic: `init`

1. Ask the user via `AskUserQuestion` (or inline if not interactive):
   - Client/brand name? (default: current workspace folder name)
   - Database location? (default: `./voc/`)
   - What sources exist? (checklist: sales calls, testimonials, YT transcripts, YT comments, FB comments, DMs, other)
   - ICP description? (paste or reference existing file)
   - Funnel stages? (default: TOFU, MOFU, BOFU)
   - Use-cases? (default: hook, content-idea, ad-copy, email-subject, email-conversion, testimonial)
2. Create `./voc/` with subfolders: `raw/sales-calls/`, `raw/testimonials/`, `raw/youtube-transcripts/`, `raw/youtube-comments/`, `raw/facebook-comments/`, `raw/dms/`, `quotes/`, `personas/`.
3. Write `voc/config.yaml` capturing the answers.
4. Do NOT delete or overwrite existing `voc/` data. If `voc/` exists, merge new config and warn.

## Routing logic: `ingest`

Accept input in any of these forms:
- List of YouTube URLs (detect `youtube.com/watch?v=` or `youtu.be/`).
- Path to a folder containing `.vtt`/`.srt`/`.md`/`.txt` files.
- Path to a single transcript file.
- Raw pasted text (ask user what source type).

For each input:
1. Determine source type (sales-call, testimonial, youtube-transcript, etc.). Ask if ambiguous.
2. YouTube URLs: invoke `scripts/fetch-youtube-transcript.sh <URL> <tmp-dir>`. It writes `.vtt` + `.info.json`.
3. VTT/SRT files: invoke `scripts/vtt-to-markdown.py --input <path> --output-dir voc/raw/<source-type>/ --source-type <type>`.
4. Raw text paste: write directly to `voc/raw/<source-type>/YYYY-MM-DD_<source>_<slug>.md` with minimal YAML frontmatter prompting the user for missing fields.
5. Report what was ingested: filenames, segment counts, inferred speakers.

## Routing logic: `extract`

1. Scan `voc/raw/**/*.md`. Build a list of raw files.
2. Scan `voc/quotes/all-quotes.jsonl`. Collect `source_file` values already extracted.
3. Compute the delta: raw files with zero or low quote coverage.
4. **Chunk the delta into batches of max 6 files per batch** (prevents agent timeouts). For this context, "timeout" means >10 minutes per Agent call.
5. Spawn parallel extraction agents via the `Agent` tool (subagent_type: `general-purpose`), one per batch. Each agent:
   - Reads its assigned raw files.
   - Extracts 8-15 high-impact verbatim quotes per file.
   - Applies the full tagging schema (speaker_type, pain_point, funnel_stage, use_for, mining_angle, confidence).
   - Appends to a per-batch JSONL in `/tmp/coaching-db-extract-<batch>.jsonl`.
6. After all agents finish, concatenate the per-batch JSONLs into `voc/quotes/all-quotes.jsonl`. De-duplicate by exact quote string + source_file.
7. Report: quotes added, distribution, speakers covered, gaps flagged.

See `references/mine-angles.md` for the default extraction angle priorities.

## Routing logic: `mine <angle>`

1. Read `references/mine-angles.md` to resolve the angle name to its lens description if the angle is recognized. If custom, use the user's angle name as the lens directly.
2. Spawn a fresh extraction agent with explicit instructions:
   - Read ALL raw files (not just the delta).
   - Apply the lens described by `<angle>`.
   - Pull quotes that this lens specifically surfaces, even if the transcript was mined under a different lens before.
   - Tag `mining_angle: <angle>` on every quote.
3. De-duplicate against existing `all-quotes.jsonl` by exact quote string + source_file.
4. Append new quotes. Report the delta.

## Routing logic: `synthesize`

1. Run `python3 scripts/build-voice-bank.py --jsonl voc/quotes/all-quotes.jsonl --output voc/personas/voice-bank.md`.
2. Report: sections written, quotes per section, gaps flagged.

## Routing logic: `status`

1. Count raw files per source type.
2. Count quotes total and per funnel stage, pain point, use_for, speaker_type.
3. Identify gaps: source types with zero files, funnel stages with <5 quotes, low-confidence quote ratio, speakers under-represented.
4. Report as a table.

## Critical design rules

- The skill runs in whatever workspace is current. The `voc/` folder is always in the project root of that workspace. The skill itself is global.
- Raw files are written ONCE. Never edited after ingestion.
- The JSONL is append-only with de-duplication on write.
- Voice bank is fully regenerable from the JSONL. Never hand-edit it.
- Per-project `voc/config.yaml` captures client-specific context so the same skill works across Precision Brass, Trombone Timo, Robinson's Remedies, and future clients.
- No em dashes anywhere in output. Hook blocks them.
