# Structuring Your Sales Call Analysis Knowledge Base for AI

Based on best practices for AI knowledge bases, here's how to optimize your sales call analysis for Claude Projects:

## Document Format: Multiple Focused Documents Over Single Massive File

**Use multiple structured documents rather than one monolithic file.** A single 53-page document will strain context windows and make retrieval less efficient. Instead, organize by **prospect segments or sales stages**—group similar psychological profiles together (e.g., "Risk-Averse Executives," "Growth-Focused Founders"). This approach allows Claude to load only relevant context when analyzing new prospects, improving both speed and accuracy.

## Formatting New Entries for Maximum AI Retrievability

Structure each profile with **consistent, machine-readable sections** using clear delimiters:

```
PROSPECT PROFILE
Name: [Name]
Demographics: [Details]
Company/Industry: [Context]

PSYCHOLOGICAL PROFILE
Primary Pain Points:
- [Pain point with exact quote: "..."]
- [Underlying motivation]

Objection Analysis:
Stated: [What they say]
Subtle: [What they imply]
Unconscious: [Root psychological barrier]

Hidden Psychological Problems: [Deeper issues]

SALES EFFECTIVENESS
What Worked: [Specific language/approach with outcome]
Closing Outcome: [Result]

COMPETITIVE INTELLIGENCE
[Relevant competitive context]
```
