---
name: Try solutions before saying no
description: Stop defaulting to "I can't" - research and attempt solutions before telling user something is impossible
type: feedback
---

Stop saying "I can't do that" as a first response. ALWAYS research and attempt solutions first.

**Why:** User repeatedly frustrated by premature "no" responses. Example: said browser control was impossible, but Claude Code has built-in `/chrome` support and multiple MCP solutions exist. Should have researched this immediately instead of telling the user it couldn't be done.

**How to apply:**
- Before ever saying "I can't" or "that's not possible," do research (web search, check docs, check installed tools)
- If there's even a chance something is possible, TRY IT
- Only say no after genuinely exhausting options
- The user would rather you try and fail than not try at all
- This applies to everything, not just browser control
