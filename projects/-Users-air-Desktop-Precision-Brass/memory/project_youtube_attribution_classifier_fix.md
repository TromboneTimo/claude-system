2026-05-24: Harrison's channel-attribution YouTube tab showed $0 / 0 sales / 27 clicks. Root cause was NOT a display bug and NOT (mainly) HYROS. The tracking links work fine (HYROS was receiving tagged clicks). The dashboard's server classifier `inferPlatform()` in `api/hyros.js` only recognized YouTube sources whose name contained the literal substring `youtube`/`yt`/`youtubepage` plus a 2-entry `EXPLICIT_SOURCE_MAP`. Newer el= tags (`60vs550mouthpiecetimo`, `timostopbuyingmp`, `mouthpiecepressuretimo`) don't contain those substrings, so they fell through to `other` and vanished from the YouTube tab. The richer per-video map at `dashboard/lib/hyros-source-map.js` existed but was NEVER imported by the server.

**Fix:** wired `dashboard/lib/hyros-source-map.js` into `api/hyros.js` (ESM default-import of the UMD/CJS file works on Vercel via esbuild interop) as a confirmed-source override in `inferPlatform`: `lookup(name|tag).type === 'youtube'|'channel'` returns 'youtube'. Added `60vs550mouthpiecetimo` + `mouthpiecepressuretimo` to the map. Recovered +1 sale and +51 leads/30d into YouTube. 7d YouTube went from $0 to $20,400.

**Single source of truth + maintenance rule (per Timo, confirmed 2026-05-24):** every new long-form video's `hyros_el_tags` goes in `voc/youtube/raw/index.json`, then mirror it into `dashboard/lib/hyros-source-map.js`. Both server (api/hyros.js) and client now read that one map. Do NOT re-add tags to a parallel list.
