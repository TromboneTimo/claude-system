Meta Ads dashboard (`dashboard/meta-ads.html`) revenue/sales come from the **`hyros-revenue` Supabase edge function** at `supabase/functions/hyros-revenue/index.ts`.

**IMPORTANT: `supabase/functions/` is GITIGNORED** in this repo, so the edge function source is NOT committed. Edits live only on disk + on the deployed function. Deploy with:
`SUPABASE_ACCESS_TOKEN=<PAT> supabase functions deploy hyros-revenue --project-ref iwlernqpwdsjarygoeog` (no Docker needed; uploads via API). PAT in [[project-credentials]].

**Attribution model (PER-AD, fixed 2026-05-26):** each sale is credited to exactly ONE specific paid Meta AD via HYROS `sourceLinkAd.adSourceId` (= Meta `ad_id`, joins to `meta_ads.ad_id`); the ad set is `adSource.adSourceId`. Pick the touch that gets credit: the paid ad that ORIGINATED the lead (`firstSource`); if first touch was not a paid ad (e.g. organic FB page) fall back to the CLOSER (`lastSource`). The function returns one record per AD (`ad_id`, `ad_set_id`, `net_collected`, `sales_count`, lines). 365d totals ~$306K across 40 ads; 30 of 40 join to meta_ads (the other ~$34K are deleted creatives, shown in the coverage banner). History: started as first-touch-only AD-SET attribution (every ad in a set showed the same set total = wrong, "all ads have 1 sale"); 2026-05-25 added last-touch fallback at set level; 2026-05-26 moved to true per-AD.

**meta_ads_daily is TRUE per-day (fixed 2026-05-26):** `meta-ads-sync` feeds `meta_ads_daily` from a `time_increment=1` insights pull (time_range incl. today), so each row = that day's spend/leads (NOT the lifetime total). Re-pulling last 30d every hourly run self-heals history (upsert on ad_id+snapshot_date). Before this, the sync wrote `date_preset=maximum` (lifetime) into the daily table, so the ad-detail day-by-day chart compared a $0.13 daily value to the $143 lifetime = 110,169%. The trend modal (`openDetail`) now shows window TOTALS + last-day + dated sparkline, NOT a per-day "% vs prior" (a single-day % is meaningless on volatile daily spend). If a day-by-day chart looks wrong again, first check meta_ads_daily values are daily not cumulative.

**Calls (added 2026-05-26):** the edge function also pulls HYROS `/calls` and attributes booked calls PER AD (same originator-else-closer logic), returning `calls_count` per ad + `totals.calls_count`. Funnel is Lead -> Booked Call -> Sale; Cost per Booked Call (spend/calls) is the high-ticket steering metric (target $40-60, per Roezan research). Table has Calls + Cost/Call columns after Lead Cost; cards add Calls/Cost-Call/Lead->Call/Call->Close/Profit. Table is now 18 cols / width 1608. Sticky Status/Ad columns were REMOVED (they sheared the table in Safari/half-cached bundles).

**Bucket grading is money-first (2026-05-26):** `classify(r)` grades winner/mediocre/bleeder by OUTCOMES, not lead cost: (1) has revenue -> ROAS (>=2 win, >=1 med, <1 bleed); (2) no sale but booking calls -> cost-per-call ($40-60 target: <=60 win, <=120 med, >120 bleed); (3) no calls/sales -> learning window=watch, $150+ poor-leads=bleeder, cheap-leads-unproven=mediocre. Cheap leads alone are NOT a winner anymore. IMPORTANT: `classify(row)` must run AFTER the row is enriched with hyros_calls/hyros_revenue (calling it on the raw meta_ads row sees 0). `rowAlerts()` matches (red on ROAS<1 / cost-per-call>120). Effect: global Winners went 29 (old cheap-lead basis) -> 2 (real money/call winners).

**Table is data-driven + auto-hiding (2026-05-26):** `tableColumns()` is the SINGLE source of truth -- each column owns its width + `cell(r,pc)` render fn + `has(r)` data test. `tableHtml` shows a column only if `always` OR some row in the current view has data (`c.has(r)`), and generates headers + `<col>` widths + `<td>`s all from that one filtered list, so they can NEVER desync (this killed the recurring column-shear bug; table width = sum of visible widths, set inline). Empty money columns auto-hide on non-converting sets; engagement columns hide on paused/no-delivery sets. CPA was REMOVED from the table (redundant with ROAS, blank ~90% in high-ticket); CPA + Hold25 + Avg Watch + CPM + full funnel breakdown live in the row-click detail modal (`openDetail` grid). To add/change a table column, edit `tableColumns()` only.

**Display:** `dashboard/meta-ads.html` builds `state.hyrosByAd[ad_id]`; each row joins by its OWN `r.ad_id` so distinct ads show distinct revenue. `state.hyrosByAdSet` (summed from per-ad) feeds only the coverage banner. Because per-ad revenue is distinct, `aggregate()` and `adSetRevenueForRows()` just SUM rows (no dedup needed). Zero shows `$0` / `0` not `.`. Metrics on table: added ROAS (revenue/spend), relabeled Sale Cost->CPA. Cards: Revenue/ROAS/Sales + CPA/AOV/Conv Rate. Verified live: in NEW Warm Ads 2026, only "P-B 0013 Hollywood Bowl" shows $6.7K/1 sale/ROAS 46.95x/CPA $142.78; other 19 ads $0. Table alignment: 16 headers=16 cols=16 cells, width 1472px.

**Why a set can legitimately show $0:** brand-new active ad sets get leads but no closed sales yet. The "All Active" tab is mostly new 2026 sets ($0); the converting revenue lives in older/paused sets (see the "All" tab). To verify visually use [[reference-view-authed-dashboard]].
