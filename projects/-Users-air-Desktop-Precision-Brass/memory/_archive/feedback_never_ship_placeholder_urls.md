**HARD RULE. No exceptions.**

Never ship a placeholder URL, fabricated link, or unverified video reference to any production destination (email_proposals, dashboard, AC, social posts, Slack notifications, anything that leaves Claude's chat window and lands somewhere a human will see).

If any link cannot be resolved to a real, working destination at the moment of insertion, the link MUST be removed entirely. Never insert a template-style placeholder like `youtu.be/PLACEHOLDER`, `[INSERT URL HERE]`, `example.com`, or a `?el=email_pp_NAME` tag where NAME points to a student without a public video. If the surrounding copy implies a link (e.g. "Watch X's full story here"), that copy ALSO gets removed when the link is removed. No orphan "watch the video" sentences with no video to watch.

**Why this exists:** caught 2026-05-17 when 9 email drafts shipped to Harrison's dashboard review queue with `youtu.be/PLACEHOLDER?el=email_pp01_benny` (and `_pp03_tony`, `_pp05_trevor`) embedded in their bodies. Harrison flagged it as a hard fail. Timo verbatim: "Under no circumstances can you allow this to happen again." The drafts had been generated by an earlier pb-email-write run that pulled real student quotes from `voc/lessons/raw/` and `voc/sales-calls/extracts/won-deals-voice-bank.md` (verified sources) but then auto-inserted a YouTube link with a literal PLACEHOLDER token because the students cited (Benny Ferguson, Tony) don't have public testimonial videos. Only the 10 students in `voc/testimonials/raw/` have public videos. The skill blindly inserted a link template regardless.

The cost when this ships: every email recipient who clicks the broken link hits a YouTube 404. They lose trust in the sender. The sender (Harrison) loses trust in the system (pb-email). The system loses Timo's trust.

**How to enforce.** Three layers:

1. **Drafting-time check:** When an agent in `pb-email-write` cites a student by name and reaches for a "watch the testimonial" link, it MUST verify the student appears in `voc/testimonials/raw/` AND extract the actual `source_url` from that file's frontmatter. If the student is found in `voc/lessons/raw/`, `voc/sales-calls/raw/`, or any voice bank but NOT in `voc/testimonials/raw/`, the agent writes the quote inline WITHOUT a "watch the video" sentence. The student's quote stays; the fabricated link does not.

2. **Auditor check (pb-email Agent 7/9):** Grep every body's `<a href="...">` for: `PLACEHOLDER`, `INSERT`, `TODO`, `XXX`, `[link]`, `example.com`, `youtu.be/PLACEHOLDER`, `precisionbrass.com` (the dead landing-page domain), and any `?el=email_pp\d+_\w+` tag where the `\w+` slug doesn't match a known public-testimonial student. Auto-reject the candidate and send it back to the originating agent. The dashboard's `email-lint.js` should ALSO enforce a subset of this at push time (any href matching the regex blocks the push).

3. **Push-time check (`pb-email-push`):** Before inserting any row to `email_proposals`, scan body + cta_url + ps_text for the same patterns above. If any hit, ABORT the push with a clear error message naming the offending URL. Do not push and re-prompt for fix.

**When this rule does NOT apply.** Memory files, knowledge files, skill references, internal documentation, and chat-only outputs that won't be shipped anywhere. Those can carry `[TODO]` markers freely.
