---
name: Skill improvement feedback loop
description: When the user gives feedback on marketing skill outputs, update the relevant skill files, SOPs, reference files, and feedback log to improve future outputs
type: feedback
---

When the user provides feedback on any marketing skill output (good or bad), take these actions:

1. Log the feedback in `references/skill-feedback-log.md` with date, skill used, what worked, what didn't
2. If positive — add the output as a best example in `references/content-examples.md`
3. If negative — update the relevant skill SKILL.md, SOP, or reference file to prevent the issue
4. Save particularly good outputs to `references/content-examples.md` as patterns for future generation

**Why:** The user wants skills to learn and improve over time, not stay static. Treating skill files as living documents that get refined with each use is the core value proposition.

**How to apply:** After every marketing task, briefly ask if the output hit the mark. When the user corrects something, immediately update the source file so it never happens again.
