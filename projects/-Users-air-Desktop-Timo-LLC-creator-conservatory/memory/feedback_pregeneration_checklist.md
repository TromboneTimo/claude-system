# Pre-Generation Checklist, Run This BEFORE Any Content

**Rule:** Before generating any hook, slide, ad, email, script, or creative output, run through this 8-item check. If any fail, STOP and fix before shipping.

**Why:** On 2026-04-13, I wrote `feedback_visual_qa.md` and `feedback_fabricated_content_numbers.md` and then violated both within the same session. The memories exist. They don't work unless I actively consult them. Plus three full deck rebuilds happened because I shipped a thesis before confirming scope.

## The 8-Item Pre-Check

### 1. Scope Locked
Can I state the ICP + scope + thesis + CTA in one sentence? If no, confirm with Timo FIRST. Don't commit to a framing without alignment.

### 2. Files Read From Disk THIS TURN
Not from memory. Use the Read tool on:
- `TIMO_PROFILE.md` (credentials, clients, business facts)
- `context/audience.md` (ICP pain, voice-of-customer)
- `context/brand.md` (positioning, proof)
- `context/offers.md` (pricing, offer structure)
- `context/LAUREL_PORTIE_DATABASE.md` (for Portie citations with line references)
- Any relevant transcript files

### 3. Citations Diversified
If citing someone, use 2+ different authorities. Never stack one source 3x (not Portie 3x). Timo's intellectual lineage: Portie, Hormozi, Gadzhi, Cialdini, Schwartz, Kevin Kelly, Seth Godin, Naval Ravikant, Greene, Kahneman.

### 4. Visual Math Done (slides)
- Will hero text at max clamp overflow the viewport?
- Is supporting text 20-25% of hero size, NOT 60-70%?
- For absolute/negative positioning: does it hold at different viewport widths?
- For quote cards: text readable, not obscured?
- Mentally simulate 11-character number at 20vw font (doesn't fit).

### 5. Credential Audit
Every credential, number, client name: does it match `TIMO_PROFILE.md` exactly?
Strip "invited/offered/nominated/considered for" (not a credential, a near-miss):
- TED talk invitation (never delivered) -> strip
- Grammy considered -> strip
- Offered a contract (not signed) -> strip
Use only things Timo DID, DELIVERED, PERFORMED, BUILT, or WON.

### 6. Layout Matches The Argument Shape
Read the slide aloud:
- "and ALSO this" -> triptych OK
- "they ALL followed the same path" -> roadmap (stage-column headers + per-instance rows + arrows)
- "first X, then Y" -> numbered timeline
- "A vs B" -> split-screen
- NEVER default to triptych for causal/sequential items.

### 7. Specific Proof, Not Generic Placeholder
If the slide is evidence (e.g., "creators with millions can't sell"), image must be a SPECIFIC documented case (PerthNow Arii headline, Obama 2008 family photo, Bing search results). NEVER use generic stock or icon placeholders for evidence slides. Generate via Gemini or fetch the actual artifact.

### 8. Bridge Slides Answer Prior Question
If slide N ends with "Why?" / "How?" / any open question, slide N+1 MUST answer it before pivoting. NEVER skip past a hanging question. Every slide needs an antecedent in the prior slide.

## When to Run This

**Mandatory before:**
- Any slide deck or HTML presentation
- Any ad creative or Power Content script
- Any email campaign or subject line batch
- Any public-facing copy for clients (Harrison, Robinson's, etc.)
- Any hook or angle output

**Not required for:**
- Internal session planning
- Priority updates
- Memory/feedback writing
- Exploratory conversations with Timo about strategy
