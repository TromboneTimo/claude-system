These rules apply to EVERY website, landing page, lead-magnet, VSL page, product page, blog skin, member portal, email-capture page, or long-form page built for Timo or his clients. Read before writing any HTML/CSS.

## 0. Ask 4-6 sharp forced-choice questions BEFORE writing any code
Never start building on assumptions. Before touching HTML, ask:

1. **Accent color?** (Red highlighter / orange / blue / other)
2. **Type voice?** (Bold punchy / editorial / quiet minimal)
3. **Primary CTA copy?** (Default "Chat with Tim" unless Timo says otherwise)
4. **Section order locked, or mirroring a reference?** (If reference, get the URL)
5. **Assets available?** (Real photos/logos vs placeholder shapes)
6. **Anti-patterns from past sites to explicitly avoid?**

Forced-choice, not open-ended. "What do you want?" wastes his time. "Red or orange?" gets an answer in two seconds.

Skipping this step and guessing costs 3-5 iteration cycles every time. The cost of one round of questions up front is always lower than the cost of rebuilding the hero, the highlight style, the CTA copy, and the section order after the fact.

**Why:** 2026-04-19 Creator Conservatory landing page. Built v1, v2, v3 on assumptions. Timo corrected each one. He said "you should ask more questions when building these sites, seems like youre really retarded." Fair. Ask first, build second.

## 1. Reference > convention > design-kit
When Timo shares a URL or screenshot, that page IS the spec. Fetch it, mirror its structure, type weight, accent treatment, and placement directives literally. Design-kit files (getdesign DESIGN.md, PlayStation/Sony guides, brand style books) are toolboxes, not commandments. When they conflict with the reference, the reference wins. Always.

## 2. Interrogate every marketing-convention default
Before reaching for a standard pattern, ask: is this serving the audience, or is it the AI/template/guru reflex? Kill these defaults unless they earn their spot:
- Small kicker/eyebrow text above hero headlines ("For [audience]")
- "Book a strategy call / discovery call / free consultation"
- Testimonial sliders or carousels
- "Unlock the secrets / revolutionary / game-changing"
- Feature grids with icon + heading + two-line body
- Footer email capture with "Join 10,000 readers"

His audience (musicians, creators, independent operators) reads these as marketing-slop signals.

## 3. Copy voice and visual voice must match
Bold punchy direct copy needs bold heavy display type (700-800). Measured reflective copy can use thinner weight. Never inherit a type voice from a design kit that fights the copy. If the copy shouts, the type shouts. If the copy whispers, the type whispers.

## 4. Smaller type, more spacing
Default instinct is to oversize hero (72-84px+). That is usually wrong. 48-64px reads comfortably on laptop. Put the energy into:
- line-height 1.75-1.85 on body
- 48-56px gap between headline and subhead
- Generous section padding (96-120px between major panels)

Breathing beats scale.

## 5. Placeholders are invisible or elegant, never labeled text
Gradient shapes, blank rings, neutral silhouettes, subtle texture. NEVER visible text letters (HB, RR, AB) inside avatar circles or image slots unless it's a designed monogram. Empty clean beats filled ugly. If you don't have the asset yet, make the slot visually empty and note where the asset goes.

## 6. Volume of social proof beats polish
Ship many testimonial blocks even with rougher copy over three highly-polished ones. His audience scans quantity first ("there are a lot of these") then quality second. Applies to case studies, client logos, screenshots, review counts.

## 7. Spatial directives are literal
"Below the video," "at the bottom," "under the VSL," "above the numbers" mean exactly that. Do not reinterpret through UX best practices or information hierarchy theory. Build what he said where he said it.

## 8. Anti-guru voice across every surface
Not just CTAs. Nav labels, body copy, form titles, button text, error states, footer, page title, meta description. If any copy reads like ClickFunnels, HighLevel, or a marketing-bro landing page, kill it and rewrite plain. Default CTA: "Book a chat with Tim" or "Chat with Tim." Never "strategy call."

## 9. On dark surfaces, key elements need color-coded glow
Not decoration. Functional. On an all-black page the audience's eye has no landmarks. Give the video a blue ring/glow, the form an orange ring/glow, key cards subtle white hairlines. Each major interactive element gets its own brand-anchored color treatment so the page navigates itself.

## 10. Pre-build the plan before writing code
Before writing HTML/CSS for any new page:
1. WebFetch the reference (if given)
2. List section order top-to-bottom
3. Identify accent treatment (highlight style, display weight, surface colors)
4. Flag voice constraints (anti-guru, no kickers, title case, etc.)
5. Name explicit placement directives from his messages

Skipping this plan costs 2+ iterations every time. Never skip.
