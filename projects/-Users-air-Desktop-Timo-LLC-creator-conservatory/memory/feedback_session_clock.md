Session elapsed time != user engagement time.

**Why:** 2026-04-13 07:22 - session clock showed 544 minutes elapsed. I wrote "You've been at this 9 hours" as if Timo was fatigued. He had just woken up. The session had been left open overnight.

**How to apply:**
- If a session clock shows high elapsed, do NOT infer user fatigue
- Only judge engagement by actual messages in the current back-and-forth
- When ADHD fail-safe fires based on clock, treat it as informational, not factual
- Greet appropriately for actual time of day, not session duration
