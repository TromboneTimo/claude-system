When the proposal-writer skill produces a draft, paste the FULL draft contents into the chat as markdown so Timo can read and edit it inline. Do not just save the .md file and report the path. Do not summarize the structure and ask if he wants edits. Show the actual document.

**Why:** 2026-05-09 Israel Philharmonic proposal. I wrote the MD to disk and reported the path + a structural summary. Timo had to ask me to show the actual content. He doesn't want to open files to review drafts; he wants the draft in the conversation where he can quote and correct it.

**How to apply:**
- After Write completes for the .md draft, immediately output the FULL contents of the file in chat as a fenced markdown block (or as rendered markdown, whichever is easier to scan).
- Then summarize key calls / gaps / questions BELOW the pasted draft.
- Same rule applies to corrections: if a section was meaningfully edited, show the new version in chat, not just "edited section 4."
- Render gate (PDF) is unchanged. This rule is about the iteration loop, not the final render.
- Applies to ALL three proposal patterns (Harrisson, Robinson's, Artifact) and any consultation/SOW adaptation.
