# Never Fabricate Week-by-Week Schedules

When writing a Delivery Timeline section in a proposal, do not invent specific weekly deliverables. Use only the sequence Timo has explicitly stated.

**Why:** I have repeatedly fabricated week-by-week timelines that included specific deliverables Timo never described (e.g., "Week 1 dashboard schema built; competitor tracking layer wired"). Timo has caught this twice now (Otto 2026-05-09, Victor 2026-05-11). His exact words on the Victor draft: "the timeline is largely hallucinated... I really need you to fix the fact that you keep hallucinating the week-by-week schedules. It's really stupid." This is not a stylistic preference. The fabricated detail commits Timo to delivery cadences he never agreed to and reads as filler.

**How to apply:**

1. **Before writing the timeline, ask: what week-by-week sequence has Timo actually stated?** If the answer is "none," do not invent one. Write an open-ended Delivery Terms paragraph instead (Otto pattern from `output/proposals/music-system-japan/`).
2. **If Timo has stated a partial sequence** (e.g., "Week 1 is the database, Week 2 is email, weeks 1-4 are setup, then webinar"), reflect exactly that. Use his level of specificity. Do not fill in undefined weeks with invented detail like "competitor tracking layer wired" or "voice model trained on Victor's existing content."
3. **Coarse phases beat fake weekly rows.** "Weeks 3-4: remaining system setup" is fine. "Week 3: System 1 - Domino statement workshop" when Timo never specified what's in week 3 is fabrication.
4. **Phase 2 sign-off trigger language must match what Timo says.** If he says "when Victor sees visible sales," do NOT write "when Victor signs off the systems are working." Different triggers; different commitments.
5. **The Delivery Timeline section is a HIGH-RISK fabrication zone.** It looks plausible because timelines have a familiar shape (Week 1 / Week 2 / Week 3 / Week 4). That same plausibility is what makes the fabrication easy to miss until Timo reads it and says "where did you get this?" Treat this section with the same skepticism as fabricated client quotes.

**What this rule does NOT prohibit.** It is fine to write a timeline using ONLY the language Timo has provided, even if that means a 3-row table where Weeks 3-4 are a single "remaining setup" cell. It is fine to ask Timo for more detail before drafting. What is prohibited is filling unknown weeks with invented specifics that look like real plans.

**Canonical incidents:** Otto MSJ proposal 2026-05-09 (fixed by switching to open-ended Delivery Terms paragraph). Victor proposal 2026-05-11 (this incident; fixed by mirroring Timo's stated Week 1 / Week 2 / Weeks 1-4 / after week 4 sequence and changing Phase 2 trigger to "visible sales generated").
