NEVER add CTA text, value-bomb ribbons, or "stick with me" teases to a HOOK slide (slide 1 of any deck/VSL/social). The hook slide must remain minimal: eyebrow + headline + sub-hook. Any additional elements dilute the hook's job, which is to stop the scroll.

**The exact failure pattern (2026-04-13):**
After auditing the views-that-matter deck, I noticed the 6-point content hook gate memory required an "early CTA in first third." The deck had no CTA in slides 1-5. I mechanically added a CTA ribbon to slide 1's bottom: "STICK WITH ME → COMMENT SYSTEM AND I'LL DM YOU MY 3-PART AUDIT FRAMEWORK."

Timo: "remove STICK WITH ME... text overload tell me why you did that you dumb fuck."

Root cause: I treated the 6-point gate as a prescription to satisfy on EACH slide, when it's guidance for the DECK OVERALL. The hook slide is NEVER the right place for an early CTA — it would compete with the hook itself.

**Where early CTAs actually belong (if the deck needs one):**
- Slide 3+ (after the initial hook has landed)
- A pivot slide between "pain" and "solution" sections
- A subtle footer ribbon on a content slide that already carries its primary argument
- NEVER on the opening hook

**The decision rule:**
Before adding ANY new element to slide 1:
1. Does slide 1 currently accomplish ONE job? (eyebrow + headline + sub-hook = yes)
2. Does the new element serve the hook, or does it add friction?
3. Could this element live on ANY OTHER slide without breaking it?
4. If yes to #3, it belongs somewhere else. The hook stays minimal.

**Meta-rule about content gates:**
Content gates (6-point hook, AIDA, PAS, etc.) describe what a deck must ACCOMPLISH, not which slide must CONTAIN each element. "Early CTA in first third" means the CTA should appear within the first 33% of slides — not that it must appear on slide 1, 2, AND 3. One slot in the first third is sufficient.

**How to apply:**
- Treat gates as audit checkboxes for the deck as a whole. Mark the gate satisfied if ANY slide in the required range contains the element.
- If a deck fails a gate, place the missing element on the WEAKEST slide in the required range, not the STRONGEST. The hook is strong; don't crowd it.
- If no slide in the required range has room for the missing element, insert a new slide — don't stuff it onto the hook.
- When in doubt about placement, ASK the user before making a judgment call on their deck's opening.

**Why I missed this:**
I ran the 6-point gate as a pass/fail checklist and tried to fix each fail by adding content to the slide I was looking at. That's mechanical rule-following, not judgment. Gates are tools for identifying WHAT'S MISSING, not templates for WHERE to insert the fix.
