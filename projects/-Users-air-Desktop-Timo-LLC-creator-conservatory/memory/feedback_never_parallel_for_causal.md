NEVER use a 3-column parallel triptych / card grid when the 3+ items share a causal pattern, sequential flow, or same underlying mechanism. Parallel columns communicate "three unrelated things in the same category." When the argument is "three instances of the same mechanism" or "one thing leads to the next," the layout MUST show arrows, columns-as-stages, or a connected roadmap.

**The exact failure pattern (2026-04-13):**
Slide 11 of views-that-matter had 3 client wins:
- Musician: 1 long-form video → coaching program → orchestra-tier revenue
- Artist: 1 reel → local news → 5 exhibitions sold out
- Professor: 1 video → 3-continent performance invitations

I laid them out as 3 parallel triptych cards. The argument was "these all demonstrate the same compounding mechanism" — but parallel cards made them look like 3 unrelated wins. Timo: "this is a terrible presentation of this information, how can you fit this into a roadmap so it's natural and visually appealing that shows how one thing leads to the next?"

The fix: horizontal roadmap with 4 stages (Input → Signal → Action → Outcome) as column headers, each creator is a row, arrows between nodes. Instantly the shared mechanism becomes visible.

**The pre-ship test for any multi-item slide:**
Before committing to a triptych/grid layout, ask:
1. Are these items PARALLEL (same category, no ordering, no shared mechanism) OR SEQUENTIAL/CAUSAL (same flow, shared pattern, one leads to the next)?
2. If I can restate the items as "X → Y → Z → outcome" and all items fit that shape, they're CAUSAL. The slide must SHOW the flow.
3. If the argument is "same mechanism, different inputs" — show the mechanism as column headers across the top, put each instance as a row below.
4. If the argument is "step 1, then step 2, then step 3" — show the steps with arrows, not as parallel cards.

**The decision rule:**
- Parallel (no causation) → triptych OK.
- Shared pattern across instances → roadmap grid with stage-column headers + per-instance rows.
- Sequential/temporal → horizontal timeline with numbered nodes + arrows.
- Comparison (A vs B) → split-screen, NOT triptych.

**Why I keep making this mistake:** Triptych is my default go-to for "I have 3 things to show." It's visually safe and easy to build. But it COSTS argument clarity when the items share meaning. "Three cards" is the easiest layout; the right layout is often harder.

**How to apply:**
- For any slide with 3+ related items, run the PARALLEL vs CAUSAL check before choosing layout.
- If the slide's purpose is to demonstrate a shared pattern, the pattern MUST be in the visual structure (arrows, column stages, connecting lines) — not just in the copy.
- The roadmap layout scaffold: column-header labels for the shared stages + one row per instance + arrows between cells + highlighted outcome column.
- When in doubt, mentally read the slide aloud. If I say "and ALSO this creator did this, and ALSO that one did that" → triptych is fine. If I say "they ALL followed the same path" → roadmap required.
